# majutsu v0.6.12 release evidence

## Release

- version: 0.6.12
- date: 2026-07-12 JST
- version output: `mj 0.6.12`

## Summary

- Prevented fanotify events for excluded, volatile, and explicitly untracked
  paths from starting unnecessary snapshots.
- Skipped follow-up remote sync and prune when a watch-triggered snapshot is a
  no-op.
- Added exponential backoff for transient SQLite `database is locked` errors in
  watch snapshot child processes. Exhausted retries are deferred without
  terminating the daemon.
- Added transient S3 response and transport retries to read, list, multipart,
  lifecycle, and archive-restore paths.
- Added operation change summaries used by history output after compacted
  snapshot metadata is unavailable.
- Added the operation-change table to the canonical database schema so fresh
  homes and migrated databases can record snapshot change summaries.
