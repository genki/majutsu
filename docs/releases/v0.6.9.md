# majutsu v0.6.9 release evidence

## release

- version: 0.6.9
- date: 2026-07-04 JST
- version output: `mj 0.6.9`

## summary

- Fixed `mj fsck --quick` false positives for valid `path-tracked` and
  `path-untracked` operation records.
- Aligned cached remote root ack refs with the current host-scoped remote
  layout by using the host name-derived remote prefix instead of the stable
  host id for those ref keys.
- Bounded `mj health --deep` remote object checks with a default 15 second
  timeout so health remains usable in unhealthy or inconsistent remote states.
- Existing encrypted states now repair age keyring permissions when common
  commands touch the state.
- `mj daemon doctor` now warns about stale systemd units that still use the
  legacy forking daemon style or lack `MemoryMax=` / `OOMPolicy=stop`.
- README and docs now consistently describe the current remote top-level
  `<host-prefix>/...` layout while retaining host id as metadata identity.
