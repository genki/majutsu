## highlights

- Reworked crates.io publishing to expose only the public root `majutsu` crate.
- Marked repository-local support crates as `publish = false`.
- Embedded support source under `src/internal/` so the published root crate has
  no dependency on unpublished private `majutsu-*` crates.
- Added a local sync/check helper for the embedded support source so release
  gates catch drift and stale mirror files from the private workspace crates.
- The sync helper discovers private support crates from Cargo metadata instead
  of keeping a hardcoded module list.
- Suppressed dead-code warning noise only at the embedded support module
  boundary, keeping normal warning checks useful while making `cargo install`
  output quieter.
- Included integration tests in the crates.io package so Cargo does not report
  auto-detected tests as ignored during package verification.
- Updated release documentation and publish helper to use the single-crate
  publishing model.
