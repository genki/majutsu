- date: 2026-06-23 JST

## summary

- Fixed Linux fanotify attribution buffering so root-owned watch daemons record
  the mutating process PID from the latest write event in a buffered batch.
- Kept Linux watch attribution strict: root-owned fanotify remains the expected
  backend for process-origin tracking, while notify/inotify modes remain
  explicit lower-trust fallbacks.
- Verified this environment records new watch operations as
  `fanotify:pid-...` and returns `mj health` to `state protected`.
