# majutsu v0.6.6 release evidence

## release

- version: 0.6.6
- date: 2026-06-29 JST
- version output: `mj 0.6.6`

## summary

- Hardened age keyring handling. `keys/recipients.toml` is now written
  atomically, restricted to 0600 on Unix, and existing keyring permissions are
  repaired when the state layout is touched.
- Made clone restore remote `keys/recipients.toml` through the same keyring
  writer so encrypted clone bootstrapping preserves key-file permissions.
- Fixed key rotation recovery fallback so `master.key.pending` and
  `master.key.prev` are decoded with the explicit fallback key file instead of
  being shadowed by `MAJUTSU_MASTER_KEY`.
- Added a file remote conditional-put fallback for filesystems where hard links
  are unavailable. The fallback uses a lock file and atomic rename, avoiding
  partial final objects.
- Updated README and feature documentation for the current host-scoped remote
  object layout and Linux fanotify default.
