# majutsu v0.6.2 release evidence

## release

- version: 0.6.2
- date: 2026-06-28 JST
- version output: `mj 0.6.2`

## summary

- Fixed `mj state` default baseline semantics. With no reference, `mj state`
  now compares each root against the first snapshot that contains that root,
  matching the documented root-added baseline behavior.
- Added `basis_roots` to `mj state --json` so multi-root default state output
  exposes the per-root baseline snapshots.
- Fixed root detection in snapshot metadata for the tree-manifest format by
  checking `root_trees` as well as legacy flat `roots`.
- Protected each root's first snapshot from `mj prune`, preventing future
  pruning from breaking default `mj state` semantics.
- Fixed `mj state` tombstone tracking so snapshot-observed temporary paths are
  not reported as deleted managed files, while explicit and journal-tracked
  deleted files remain visible.
- Restored the `tracked_paths.tracking_source` schema/migration definition in
  the support crate so fresh state homes can record explicit/journal tracking
  sources correctly.
- Updated prune/GC tombstone E2E coverage for the first-snapshot protection
  rule: the first root snapshot remains available as the default state baseline
  while middle history snapshots can still be pruned and tombstoned.
