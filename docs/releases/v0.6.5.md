# majutsu v0.6.5 release evidence

## release

- version: 0.6.5
- date: 2026-06-29 JST
- version output: `mj 0.6.5`

## summary

- Hardened clone/materialize and restore hydrate paths against remote metadata
  object-key traversal. Local object keys now reject absolute paths, `..`, `.`,
  empty components, backslash-separated keys, and non-`objects/` local object
  keys before touching the local state home.
- Added clone metadata validation for snapshot, blob, large object, chunk, pack,
  and pack-index object keys.
- Made file remote writes atomic. Conditional puts now write to a unique temp
  file, fsync it, and publish with an atomic hard-link step so failed writes do
  not leave a partial final object that future uploads skip.
- Made `config.toml` and master-key writes atomic.
- Changed second-based duration parsing so non-zero sub-second strings such as
  `500ms` round up to one second instead of silently disabling periodic rescan.
- Improved encrypted key rotation crash tolerance by writing pending/previous
  key files, creating newly encrypted objects before switching the active master
  key, and allowing decode fallback to pending/previous keys during recovery.
