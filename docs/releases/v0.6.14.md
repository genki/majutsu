# majutsu v0.6.14 release evidence

## Release

- version: 0.6.14
- date: 2026-07-16 JST
- version output: `mj 0.6.14`

## Summary

- Made `mj status` read-only by default so status and diagnostic commands do not
  start a stopped watch daemon. `mj status --start-daemon` is the explicit opt-in.
- Added regression coverage for `fsck --quick`, `sync status`, `remote check`,
  and dry-run remote repair preserving a stopped daemon state.
- Added `mj remote migrate-legacy --host <id-or-name> [--dry-run]` for a
  non-destructive migration from pre-host-prefix remote layouts.
- Migrated S3 metadata uses the current compressed metadata key and canonical
  content representation. The old remote data is retained until clone/restore
  verification is complete.
- Documented the migration procedure and recorded the GitHub issue findings.
