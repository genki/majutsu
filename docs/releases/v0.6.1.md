# majutsu v0.6.1 release evidence

## release

- version: 0.6.1
- date: 2026-06-27 JST
- version output: `mj 0.6.1`

## summary

- Hardened S3-compatible E2E validation for the current direct host-prefix
  layout: remote objects must live under `<host-label>/...`, and old unprefixed
  payload or global metadata paths are rejected.
- Fixed remote recovery paths that still assumed older object layouts when
  hydrating compact snapshot manifests, tree nodes, canonical aliases, chunk
  indexes, encrypted key metadata, and GC marks.
- Kept default sync payload-cache pruning while leaving metadata cache available
  for responsive CLI commands unless `MAJUTSU_SYNC_LOCAL_METADATA_CACHE_PRUNE=1`
  is explicitly set.
- Tightened include/exclude precedence: broad include globs no longer pierce
  excluded directories, while explicit paths and direct file-pattern opt-ins can
  still recover intentionally tracked files.
