# majutsu v0.5.10 release evidence

## release

- version: 0.5.10
- date: 2026-06-24 JST
- version output: `mj 0.5.10`

## summary

- Prevented `database is locked` failures in read-like CLI commands such as
  `mj log` while daemon snapshot/sync work is active.
- Added a SQLite busy timeout for every majutsu DB connection.
- Migrated the local SQLite database to WAL mode so readers are not blocked by
  normal writer activity.
- Batched `tracked_paths` updates into the snapshot commit transaction instead
  of issuing per-file autocommit writes during root scans.
- Added a regression test that holds a writer lock and verifies `mj log` waits
  instead of failing immediately.

## local validation

```sh
cargo fmt --all --check
cargo check --locked
cargo clippy --locked --all-targets -- -D warnings
cargo test --locked --workspace
```

- result: pass

## runtime validation

After installing the fixed local binary and restarting `majutsu.service`, repeated
`mj log` calls during daemon startup/snapshot activity no longer failed with
`database is locked`. The state database was migrated to WAL and produced
`majutsu.sqlite-wal` / `majutsu.sqlite-shm`.
