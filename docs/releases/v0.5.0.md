## highlights

- Added opt-in tree manifest v2 with `MAJUTSU_TREE_FORMAT=v2`.
- Changed v2 tree storage to hierarchical Merkle directory nodes.
- Reused unchanged subtree node keys across snapshots.
- Reworked tree node generation to build a directory trie before storing nodes.
- Moved restore planning, diff, and fsck payload traversal onto direct node
  traversal paths instead of requiring full flat tree expansion.
- Kept v1 flat tree manifests readable for existing timelines.
- Left v2 writing opt-in until production-like GCS measurements confirm object
  count, metadata size, request count, and restore latency behavior.
- Added root size, traffic, RSS, and corrupt summary regression coverage from
  the 0.4.x performance work.
