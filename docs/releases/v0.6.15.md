# majutsu v0.6.15 release evidence

## Release

- version: 0.6.15
- date: 2026-07-17 JST
- version output: `mj 0.6.15`
- build number: 25

## Summary

- Improved the default `mj remote fsck` path for S3 compact-head remotes.
- Quick metadata fsck now treats the current compact head and GC mark as
  authoritative and avoids downloading the complete historical tombstone set.
- Full historical tombstone validation remains available through deep metadata
  verification.
- Added a default 60-second quick-fsck timeout and phase progress on stderr.
- Added regression coverage for progress, timeout behavior, and the MinIO
  compact-head path.

## Validation

GitHub Actions are not used; release validation was performed locally.

cargo fmt --all -- --check

cargo clippy --workspace --all-targets --locked
RUST_TEST_THREADS=1 cargo test --workspace --all-targets --locked -- --nocapture
scripts/e2e-minio.sh

- Workspace tests passed.
- MinIO via Podman E2E passed, including multipart upload and clone/restore
  byte comparisons.
- Actual GCS remote metadata fsck passed for `vagrant`, `mba22`, and `winvr`
  in approximately 10 seconds.
- Sample object verification passed with 20/20 objects present.
